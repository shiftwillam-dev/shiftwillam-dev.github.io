Data quality principles

Missing values are not silently imputed away. SEER-specific codes such as Blank(s), Unknown, and special tumour-size codes are standardized and represented with explicit indicators for downstream modelling.

No silent deletionRare ages and large tumor sizes are flagged for review instead of being removed automatically.
No leakage by accidentSurvival time and vital status are kept for audit but excluded from prediction features.
Missingness is signalClinical unknowns are represented as explicit categories or flags.
Temporal realismThe split follows diagnosis time so the evaluation resembles future deployment.

Preprocessing log

Open each transformation to explain the cleaning decisions

The full registry export is reduced to variables that support the diagnosis-time prediction task: demographics, tumor characteristics, staging, biomarkers, lymph node status, metastasis indicators, and outcome audit fields.
The pipeline records a checksum and basic dataset profile so future runs can detect accidental changes before any analysis begins.
Long SEER labels with spaces, punctuation, and parentheses are converted to consistent snake_case names for safer modelling code.
A stable row identifier is created while retaining the original patient identifier for deduplication and audit checks.
For patients with multiple breast cancer records, the earliest diagnosis is retained to avoid patient-level leakage between training and evaluation periods.
SEER cause-specific death classification is mapped to `cancer_specific_death`, where 1 means death attributable to breast cancer and 0 means alive or death from another cause.
Rows with missing or unknown cause of death are excluded from target modelling because assigning them to either class would introduce label noise.
Age recodes are converted into numeric age while preserving the top-coded 90+ indicator as a separate flag.
True tumor-size measurements are converted to millimeters, while administrative special codes are treated as missing-like values with explicit flags.
Income categories are converted into approximate numeric midpoints while preserving missing and top-coded information.
Different SEER missingness labels are standardized so modelling and visualization code can handle them consistently.
The dataset is labelled for temporal validation and for COVID-era sensitivity analysis, supporting a realistic deployment-style evaluation.
Flags are created for clinically important fields such as tumor size, grade, stage, ER, PR, HER2, and subtype because missingness can itself carry signal.
Rare ages and large tumor sizes are flagged for sensitivity analysis rather than removed, because they may represent real clinical presentations.
The final dataset, schema, exclusion summary, and data-quality report are produced so downstream EDA and modelling are reproducible.
