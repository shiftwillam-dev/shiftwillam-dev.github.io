Submitted BioFusion fields

    This is a course demo estimator based on project findings. It is not a medical device, does not replace clinician judgement, and must not be used for real clinical decisions.

    1

    Input

    The user enters diagnosis-time variables: age, stage, subtype, grade, race, tumor size, nodes, income context, and metastasis sites.

    2

    Risk score

    The BioFusion API returns a probability and assigns low, medium, or high risk using documented thresholds.

    3

    Request summary

    The result panel lists the mapped API fields sent to the backend so the request can be inspected.

    Deployment architecture

    The page calls a backend API instead of calculating locally

    FrontendGitHub Pages hosts the static M1-M6 website.
    Form inputThe M6 page collects diagnosis-time features from the user.
    API callThe browser sends BioFusion JSON to `/predict` on the backend service.
    Model layerThe backend computes the survival-risk probability.
    JSON responseThe API returns `probability` and `risk_level`.
    DisplayThe page updates the risk meter, request summary, and status badge.
